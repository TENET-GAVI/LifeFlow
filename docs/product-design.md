# LifeFlow 生活节律 App 设计稿

## Goal

做一个 iPhone 应用，帮助用户把生活拆成可执行、可记录、可复盘的日常节律。第一版不追求复杂管理，而是让用户每天愿意打开，并能在 1 到 3 分钟内完成计划和复盘。

## Users / Callers

- 想改善作息、学习、运动、情绪和环境的人。
- 对传统打卡 App 有压力，容易因为中断连续天数而放弃的人。
- 希望通过每日反馈逐步了解自己生活模式的人。

## In Scope

- 今日页：展示今日三件事、今日习惯、完成度和复盘入口。
- 习惯页：创建和查看习惯，显示目标、时间段、稳定度。
- 复盘页：记录状态、保留事项、明日重点。
- 趋势页：展示最近 7 天完成率、30 天生活维度稳定度。
- 本地数据优先：第一版不需要登录。

## Out Of Scope

- 社区、好友 PK、复杂成就系统。
- 多设备同步和账号系统。
- Apple Watch、Health、日历接入。
- 复杂 AI 对话。第一版只保留未来可接入总结能力的结构。

## Behavior

### 核心闭环

1. 早晨打开 App，看今天三件事和需要执行的习惯。
2. 白天完成习惯，点按切换完成状态。
3. 晚上完成 3 分钟复盘。
4. 趋势页根据最近数据反馈稳定度和薄弱环节。
5. 下一天根据复盘调整任务强度。

### 信息架构

- 今日：默认首页，也是使用频率最高的页面。
- 习惯：管理长期行动。
- 复盘：记录当日状态和明日重点。
- 趋势：观察生活维度变化。

### 视觉方向

- 关键词：安静、清醒、有秩序、低压力。
- 主色：纸白、墨绿、薄荷绿。
- 辅助色：天空蓝、珊瑚红、暖金、叶绿。
- 交互语气：不评价、不惩罚，只提醒用户“下一步是什么”。

## Edge Cases

- 用户没有完成习惯：允许跳过，后续可记录原因。
- 用户连续几天没有打开：回到今天，不展示失败惩罚。
- 用户任务过多：今日页只显示三件事，避免过载。
- 用户只想快速复盘：复盘页应支持一分钟内提交。

## Acceptance Criteria

- 用户进入 App 后，首屏能立刻看到今天要做什么。
- 用户能点按习惯完成状态，并看到完成度变化。
- 用户能打开晚间复盘弹层，选择情绪并保存。
- 用户能在底部 Tab 中切换今日、习惯、复盘、趋势。
- 趋势页能展示最近 7 天完成率和生活维度稳定度。
- 原型在桌面和手机宽度下都不出现文字重叠。

## Verification

- 打开 `prototype/index.html` 查看静态高保真原型。
- 后续如果转 iOS 工程，使用 SwiftUI 复刻当前信息架构和视觉系统。

## iOS Implementation Notes

- UI：SwiftUI。
- 数据：SwiftData，先本地存储。
- 通知：UserNotifications，用于习惯提醒和晚间复盘。
- 图表：Swift Charts。
- 后续同步：iCloud / CloudKit。
- 后续智能总结：先基于本地规则生成周总结，再接入 AI。
